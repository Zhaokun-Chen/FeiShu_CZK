# 会议全链路伴侣 — 详细工作流

## 会前简报工作流

### 输入
- 会议主题（如"Q2 产品规划评审"）
- （可选）会前准备文档的 docx_token

### 处理步骤
1. **主题规范化**
   - 去除日期、"智能纪要"前缀等噪音
   - 生成检索 query

2. **相关文档检索**
   - 调用 `lark-cli docs +search --query <主题>`
   - 最多返回 5 篇相关文档
   - 排除当前文档本身

3. **摘要生成**
   - 读取每篇相关文档的前 200-400 字
   - 提取与会议主题相关的片段

4. **历史行动项关联**
   - 在历史追踪 Base 中搜索与主题相关的未完成任务
   - 最多返回 5 条

5. **当前会议速览**
   - 提取文档中的"总结""待办""赛事安排"等区块

6. **渲染简报**
   - 生成 Markdown 格式简报
   - 调用 `lark-cli drive +import` 创建飞书文档

### 输出
- 会前简报飞书文档 URL

## 会后追踪工作流

### 输入
- 会议纪要文档 docx_token 或完整 URL

### 处理步骤
1. **文档读取**
   - 调用 `lark-cli docs +fetch --doc <docx_token>`
   - 获取原始 HTML 和清理后的纯文本

2. **智能纪要检测**
   - 检查文档是否包含 `<h1>待办</h1>`、`<checkbox>`、`<h1>智能章节</h1>` 等标记
   - 如果是智能纪要 → 解析 `<checkbox>` 提取 AI 待办
   - 如果不是 → 走规则抽取

3. **行动项规范化**
   - 推断负责人（正则匹配"xxx负责"）
   - 推断截止时间（匹配日期表达式）
   - 标记"需确认"字段

4. **知识链接补充**
   - 基于任务内容搜索相关文档
   - 最多补充 3 个相关链接

5. **Base 写入**
   - 查找或创建"行动项追踪"表
   - 幂等写入：已有任务更新，新任务插入

6. **分发稿生成**
   - 渲染 Markdown 汇总表
   - 调用 `lark-cli drive +import` 创建飞书文档

7. **负责人清单生成**
   - 按负责人分组
   - 每人一份独立执行清单文档

### 输出
- 行动项追踪 Base URL
- 分发稿文档 URL
- 负责人清单文档 URLs

## 定时扫描工作流

### 输入
- 队列 Base（维护"待处理会议纪要"表）

### 处理步骤
1. **读取队列**
   - 扫描状态为"待处理"或"失败"的记录

2. **逐条处理**
   - 标记为"处理中"
   - 调用会后追踪工作流
   - 成功 → 标记"已完成"，回填结果链接
   - 失败 → 标记"失败"，记录错误信息

3. **循环扫描**
   - `--once`：单次扫描后退出
   - `--loop --interval 300`：每 5 分钟扫描一次

## 消息推送工作流

### 触发条件
- `--send-msg` 参数开启
- `--attendees` 指定了接收人

### 处理步骤
1. **用户搜索**
   - 调用 `lark-cli contact +search-user --query <姓名>`
   - 获取 open_id

2. **消息发送**
   - 调用 `lark-cli im +messages-send`
   - 发送文本或 Markdown 卡片消息

## 权限清单

| 功能 | 所需 Scope |
|------|-----------|
| 读取文档 | `docs:document:readonly` |
| 搜索文档 | `search:docs:read` |
| 创建/读取 Base | `base:app:readonly`, `base:app` |
| 导入文档 | `docs:document:import` |
| 发送消息 | `im:message:send` |
| 搜索用户 | `contact:user:readonly` |
| 读取妙记 | `minutes:minutes:readonly`, `minutes:minutes.artifacts:read` |
| 读取会议纪 | `vc:note:read` |
