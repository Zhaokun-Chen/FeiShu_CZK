# Demo 演示脚本

## 演示目标

用一条本地命令证明：

> Agent 可以在本地运行，但操作飞书中的真实文档、真实 Base 和真实分发稿。

## 演示前准备

- 已安装并配置 `lark-cli`
- 已完成用户身份登录授权
- 已有一篇可读取的飞书会议纪要文档

## 演示输入

测试文档：

`FIA5d3Vc3oI9QyxicMBcqBjlnAf`

## 演示流程

### Step 1: 介绍场景

说明这是一个“会后行动项与知识分发 Agent”。

输入是一篇飞书会议纪要，输出是：

- 飞书 Base 行动项追踪表
- 飞书分发稿文档

### Step 2: 展示本地命令

运行：

```bash
python agent.py --docx FIA5d3Vc3oI9QyxicMBcqBjlnAf
```

如果希望复用已有 Base，避免每次演示都新建一个库：

```bash
python agent.py --docx FIA5d3Vc3oI9QyxicMBcqBjlnAf --base-token <已有BaseToken>
```

强调：

- Agent 在本地跑
- 但读取和写入的是飞书中的真实资源

### Step 3: 解释执行过程

命令会自动完成：

1. 读取飞书会议纪要正文
2. 抽取行动项
3. 推断负责人、截止时间和状态
4. 创建 Base 并写入记录
5. 生成分发稿

### Step 4: 展示终端结果

重点看这几个输出：

- `Document`
- `Action items`
- `Created`
- `Updated`
- `Needs confirmation`
- `Base URL`
- `Distribution doc`
- `Local report`

### Step 5: 打开 Base

展示 Base 中的字段：

- 任务
- 负责人
- 截止时间
- 截止说明
- 来源会议
- 背景知识
- 状态

说明它已经从纪要变成了可追踪的任务资产。

如果是重复运行同一篇文档，还可以说明：

- Demo 会优先复用同名 Base / 指定 Base
- 相同任务会优先更新已有记录，而不是无限重复创建

### Step 6: 打开分发稿

展示飞书文档中的分发稿。

强调这是一个可直接转发给项目组的产物，不需要再人工整理一次。
可以顺带展示新增的：

- 概览区
- 行动项总数
- 待确认项列表
- 每条任务的背景依据

## 演示话术

可以直接使用下面这段：

“这个 Agent 当前先跑在本地，但它调用的是飞书官方 CLI 和飞书真实数据。给它一篇会议纪要后，它会自动抽取行动项，补齐负责人和截止时间，写入飞书 Base，再生成一篇可分发的飞书文档。也就是说，它不是停留在问答层，而是把会后纪要直接转成可执行的协作资产。”  

“为了方便演示和后续迭代，这一版还支持复用已有 Base，同一条任务重复执行时优先更新而不是反复插入，同时会在本地导出一份 JSON 报告，便于检查抽取结果和追踪问题。”  

## 可展示的结果点

- 从非结构化纪要到结构化行动项
- 从信息读取到任务落库
- 从会后整理到知识分发
- 本地运行，但飞书内真实产出

## 演示后的延伸说明

可以补一句：

“当前版本已经验证了最小闭环，下一步可以把这个本地 Agent 包装成飞书里的正式助手入口，例如机器人、Agent 或事件触发工作流。”
