# 效果验证方案 V1

## 1. 目的

这份文档用于回答比赛里最关键的一个问题：

> 这个 Agent 不是“能跑”就够了，还需要证明它真的有价值。

因此验证目标分为三类：

- 内容准确
- 产物可用
- 相比传统方式更高效

## 2. 验证对象

当前项目主要有两类知识产物：

- 会后行动项分发与追踪结果
  - Action Items
  - 负责人
  - 截止时间
  - 背景说明
  - Base 落库结果
- 会前背景简报
  - 当前会议速览
  - 会后可能延续的行动项
  - 历史相关行动项
  - 相关资料列表

## 3. 测试样本设计

建议至少准备 6 到 10 份样本，覆盖不同风格：

- 样本 A：结构化待办明显的会议纪要
- 样本 B：长篇妙记，待办分散在总结和后续工作计划中
- 样本 C：多项目混合讨论的同步会纪要
- 样本 D：运营 / 活动类会议纪要
- 样本 E：项目复盘类纪要
- 样本 F：没有明显待办区块，但含有大量“需提交 / 需完成 / 计划”表达的文档

如果时间足够，再加：

- 样本 G：存在歧义负责人和相对时间表达
- 样本 H：会前 briefing 需要结合历史任务理解背景的会议

## 4. 指标设计

### 4.1 会后链路准确性

建议统计：

- `任务抽取准确率`
  - 正确抽出的任务数 / 人工标注任务总数
- `任务抽取精确率`
  - 正确抽出的任务数 / Agent 抽出的任务总数
- `负责人识别准确率`
  - 负责人正确的任务数 / 需要负责人字段的任务数
- `截止时间识别准确率`
  - 截止时间正确的任务数 / 需要截止时间字段的任务数
- `来源可追溯率`
  - 能回溯到纪要原文依据的任务数 / Agent 抽出的任务总数

### 4.2 会前链路有效性

建议统计：

- `相关信息命中率`
  - briefing 中被评估者认为“有帮助”的要点数 / briefing 总要点数
- `历史行动项相关率`
  - briefing 中列出的历史任务里，真正与当前会议相关的比例
- `资料空跑率`
  - 生成 briefing 但未提供任何有效额外背景的比例

### 4.3 效率提升

建议统计：

- `人工整理行动项耗时`
- `Agent 自动整理耗时`
- `人工准备会前背景耗时`
- `Agent 生成 briefing 耗时`
- `会后分发准备耗时缩短比例`

一个简单的效率指标可以是：

```text
效率提升 = (人工耗时 - Agent耗时) / 人工耗时
```

## 5. 标注方式

建议采用人工标注作为第一版基准。

每份样本至少做两份内容：

- `人工标准答案`
  - 正确任务列表
  - 正确负责人
  - 正确截止时间
  - 正确背景依据
- `Agent 输出结果`

然后逐项对比。

## 6. 建议的记录表

可以维护一个本地表或飞书 Base，字段建议如下：

- `样本名称`
- `文档链接`
- `人工任务数`
- `Agent任务数`
- `正确任务数`
- `任务精确率`
- `任务召回率`
- `负责人准确率`
- `截止时间准确率`
- `可追溯率`
- `会前briefing是否有帮助`
- `人工耗时`
- `Agent耗时`
- `备注`

## 7. 最低可交付验证

如果比赛时间有限，建议先完成一版最小验证：

- 选 5 份真实纪要
- 对每份人工标注任务清单
- 统计任务精确率、召回率、负责人准确率、截止时间准确率
- 选 2 到 3 份会议做会前 briefing 评估
- 记录人工准备背景资料与 Agent 生成 briefing 的耗时对比

## 8. 结果呈现建议

答辩时不要只说“效果不错”，建议直接展示：

- 一张会后抽取准确率表
- 一张会前 briefing 有效性表
- 一张效率对比表

例如：

| 指标 | 结果 |
|---|---|
| 任务抽取精确率 | 82% |
| 任务抽取召回率 | 76% |
| 负责人识别准确率 | 71% |
| 截止时间识别准确率 | 79% |
| 会后整理耗时缩短 | 63% |
| 会前背景准备耗时缩短 | 52% |

## 9. 当前版本的已知偏差

当前 V1 / V2 / 会前 briefing 版本，已知偏差包括：

- 规则抽取可能把“赛事说明”误识别成任务
- 负责人推断仍有启发式成分
- 相对时间表达仍不稳定
- `docs +search` 召回弱时，会前 briefing 更依赖当前会议文档和历史 Base

这些偏差都应在验证报告中明确说明，而不是回避。

## 10. 当前阶段结论

比赛里的“效果验证”不需要一开始就非常学术，但必须做到：

- 有样本
- 有人工基准
- 有明确指标
- 能量化说明 Agent 比传统人工整理更省时间、更利于信息对齐
