# 项目说明

## 项目名称

会后行动项与知识分发 Agent

## 一句话定义

输入一篇飞书会议纪要或妙记总结，Agent 自动抽取行动项，补齐负责人和时间信息，写入飞书 Base，并生成可直接转发的分发稿。

## 目标用户

- 项目负责人
- 会议组织者
- 参会成员

## 要解决的问题

企业会后最常见的问题不是“找不到纪要”，而是：

- 行动项藏在长纪要里，不容易被快速执行
- 负责人和截止时间不统一
- 会后分发需要人工整理
- 后续跟进缺少统一追踪入口

## 为什么不是普通搜索或问答

普通搜索和问答只能帮助用户“找到信息”或“回答问题”，但不能直接把纪要转化成可执行、可追踪、可分发的工作资产。

本项目的核心价值在于：

- 从非结构化会议纪要中提取结构化任务
- 保留来源会议和背景知识，降低误解风险
- 自动生成追踪表和分发稿，减少人工整理成本

## 当前 MVP 能力

本地命令：

```bash
python agent.py --docx <飞书文档ID>
```

可选增强参数：

```bash
python agent.py --docx <飞书文档ID或链接> --base-token <已有BaseToken> --report-json tmp/last_run.json
```

执行后自动完成：

1. 读取飞书会议纪要文档
2. 抽取行动项
3. 标准化负责人、状态和截止时间
4. 创建飞书 Base 并写入记录
5. 生成飞书分发稿文档
6. 输出 Base 链接和分发稿链接
7. 导出一份本地 JSON 报告，便于演示和排查

## 当前验证结果

以《智能纪要：飞书AI校园挑战赛内容分享 2026年4月23日》为测试输入，当前版本已经可以：

- 本地读取飞书文档正文
- 抽取 7 条行动项
- 将 7 条行动项写入飞书 Base
- 生成飞书分发稿文档
- 将 `Needs confirmation` 压到 0
- 支持复用已有 Base，并对同任务优先更新而不是重复插入

## 技术结构

- 本地执行层：Python
- 飞书执行层：官方 `lark-cli`
- 数据源：飞书文档 / 妙记
- 输出载体：飞书 Base / 飞书文档

## 当前边界

- 不自动群发消息
- 不读取外部群成员
- 不直接创建飞书任务
- 不做复杂权限编排
